# 「有谱」开发部署说明

## 1. 系统边界

用户端和运营端都不直接读写数据库：

| 入口 | 服务端 | 权限边界 |
|---|---|---|
| 微信小程序 | `youpuUserApi` | 微信登录用户 + 家谱角色 |
| 运营管理端 | `youpuOpsApi` | CloudBase Auth v2 登录 + `operators` 白名单 |
| 定时器/审核回调 | `youpuJobs` | 禁止客户端调用，仅平台触发或带部署密钥的内部调用 |

旧 `familyFunctions` 只保留在并行切换期。生产稳定后应用 `deployment/security/function-rules-after-stabilization.json` 禁止其调用，再按变更流程下线。

## 2. 环境

- 生产：`cloud1-d5gs5yj4l283d9c6d`。
- 预发布：必须在 CloudBase 控制台新建，不能复用生产。
- 小程序环境开关：`miniprogram/config/env.js`。
- 管理端环境：构建时通过 `VITE_CLOUDBASE_ENV` 注入；缺失时生产构建会失败。

预发布期间将 `active` 设为 `staging` 并填入预发布环境 ID。上传正式版前再切回 `production`。不要用脚本替换生产 ID，也不要把密钥写进仓库。

## 3. 运行时与依赖

三个新云函数统一使用 `Nodejs20.19`，版本、超时和内存以 `deployment/cloudbaserc.example.json` 为基准：

| 云函数 | 超时 | 内存 |
|---|---:|---:|
| `youpuUserApi` | 15 秒 | 512 MB |
| `youpuOpsApi` | 20 秒 | 512 MB |
| `youpuJobs` | 60 秒 | 512 MB |

业务请求目标仍为 5 秒内；告警阈值见正式发布运行手册。依赖版本已锁定在各函数 `package.json` 和管理端 `pnpm-lock.yaml` 中。

## 4. 数据初始化

部署 `youpuJobs` 后，在云函数控制台手工测试中调用一次：

```json
{
  "action": "system.bootstrap",
  "secret": "<BOOTSTRAP_SECRET>",
  "initialOperatorAuthUid": "<Auth v2 用户 UID>",
  "initialOperatorEmail": "<运营邮箱>",
  "initialOperatorName": "<显示名称>"
}
```

它会幂等创建集合、写入 schema v3，并且只在没有有效运营账号时创建第一个 `super_admin`。之后立即轮换部署密钥，不要在日志、文档或聊天记录中保存明文。

核心集合包括：`users`、`families`、`family_memberships`、`user_family_preferences`、`persons`、`relations`、`invitations`、`change_requests`、`media_assets`、`moderation_tasks`、`reports`、`account_deletion_requests`、`idempotency_records`、`rate_limits`、`operators`、`audit_logs`、`profile_sync_tasks` 和 `backup_manifests`。

## 5. 索引与安全规则

1. 按 `deployment/database-indexes.json` 在控制台创建索引。唯一索引和稀疏属性必须逐项确认成功，不能只创建普通索引替代。
2. 数据库应用 `deployment/security/database-deny-all.json`。
3. 存储应用 `deployment/security/storage-private-staging.json`，只允许登录用户向自己的 `staging/` 暂存路径上传 5 MB 内的 jpg/jpeg/png/webp；客户端读始终关闭。
4. 云函数先应用 `deployment/security/function-rules.json`，稳定后应用 `function-rules-after-stabilization.json`。
5. 用三个测试账号分别验证 `admin/member/viewer`，并额外验证未登录、跨家谱和非运营白名单访问全部被拒绝。

同时为存储 `staging/` 前缀配置 24 小时生命周期清理。客户端在上传完成前中断时，云函数可能拿不到最终 fileID；生命周期规则是这类暂存孤儿的最后清理边界。

索引清单是可审查的部署契约；不同 CloudBase 控制台版本不保证支持直接导入，实际结果以控制台索引状态为准。

## 6. 内容安全与媒体

- `CONTENT_MODERATION_MODE` 必须为 `strict`。
- 文本在写入业务数据前同步审核；可疑文本进入 `moderation_tasks`。
- 图片先上传私有暂存目录，调用 `media.complete` 后异步审核。只有 `approved + active` 资源可通过 `media.getUrls` 得到短期 URL。
- 在微信/CloudBase 控制台配置 `security.mediaCheckAsync` 回调到 `youpuJobs`，再用通过、疑似、违规三张测试图验证状态分别进入 `approved/review/rejected`。
- 审核回调未验证前，不得开放正式图片上传。

## 7. 运营后台

1. 在 CloudBase Auth v2 开启邮箱密码登录，关闭公开注册入口。
2. 由控制台创建初始账号，再通过 bootstrap 写入 `operators` 白名单。
3. 构建：`VITE_CLOUDBASE_ENV=<环境ID> pnpm --dir admin run build`。
4. 将 `admin/dist` 部署到该环境的静态托管。
5. 配置静态站点域名/CORS，仅允许受控运营域名调用。

`operator` 可以处理举报和复核；`super_admin` 额外管理运营账号、冻结与注销重试。运营端不能修改家谱人物关系。

## 8. 状态与数据生命周期

- 家谱：`active / frozen / archived / deleting`；归档 30 天后异步清理，可在到期前恢复。
- 邀请：`active / revoked / expired / exhausted`；默认 30 天、最多 50 次。
- 媒体：`pending / approved / rejected / review`；先审后展示。
- 注销：`pending / cancelled / processing / completed / failed`；7 天冷静期，之后匿名化账户，保留由家庭共同维护的人物资料。
- 举报：`open / processing / resolved / rejected`。

所有关键写操作使用请求幂等键；家谱、关系、审核、邀请、角色、归档和注销状态变更使用数据库事务。列表使用游标分页，图谱接口硬限制为 500 人并明确报错，不做静默截断。

## 9. 部署命令

先运行正式预检，再部署预发布：

```bash
TARGET_ENV_ID=<预发布环境ID> ./deployment/preflight.sh
./uploadCloudFunction.sh <预发布环境ID>
```

微信 CLI 只能列出环境和部署云函数，不能创建环境、配置索引/规则、备份数据库或发布审核。因此这些步骤必须在 CloudBase/微信公众平台控制台完成，并把截图、环境 ID、备份恢复结果和验收人记录进发布单。

生产部署、备份恢复和回滚顺序见[正式发布运行手册](./正式发布运行手册.md)。
