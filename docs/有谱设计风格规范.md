# 「有谱」设计风格规范

> 设计方向：现代东方、温暖克制、关系清晰、全年龄友好
> 适用范围：微信小程序用户端、分享卡片、运营管理后台

## 1. 核心设计定义

「有谱」不是仿古族谱，也不是年轻化社交应用，而是一件全家都能使用的现代家庭工具。

整体风格定义为：

> **现代东方家庭图谱**——用东方文化的温度表达家族传承，用现代数字产品的清晰度承载复杂关系。

设计首先服务于“看清人和关系”，品牌和装饰退居其次。页面应该让年轻人觉得简洁，让年长用户觉得亲切，让家庭资料显得可靠而不是娱乐化。

### 1.1 四个关键词

| 关键词 | 设计含义 |
|---|---|
| 温暖 | 有家庭感，但不使用大面积怀旧棕色和泛黄纸张 |
| 可信 | 信息层级稳定、操作可预期、隐私状态明确 |
| 清晰 | 图谱、按钮、文字和状态在第一眼可以辨认 |
| 有生命力 | 家谱像持续生长的家庭网络，而不是静态档案 |

### 1.2 不采用的风格

- 不做卷轴、宣纸、木纹、牌位、祠堂式仿古界面。
- 不使用大面积金色、深棕色和复杂传统纹样。
- 不用粉色/蓝色区分女性和男性。
- 不使用大量 emoji 充当正式功能图标。
- 不做卡通、游戏化或过度活泼的家庭社交风格。
- 不追求“炫酷科技树”，避免发光线条、黑色背景和复杂 3D 动效。

## 2. 品牌视觉

### 2.1 品牌气质

品牌人格是一位可靠、耐心、懂家庭关系的整理者。它不替家庭下结论，而是帮助家人把信息理清楚、保存好、传下去。

推荐品牌表达：

- 产品名：有谱
- 品牌主张：一家人，共修一份家谱
- 功能型短句：从一个人开始，把一家人连起来

### 2.2 Logo 方向

Logo 建议把“谱页”和“关系枝条”组合为一个简洁图形：

- 外轮廓像一页册子或方形印记，表达“有记录、有依据”。
- 内部由 3—5 个圆点和连接线组成，表达家庭成员与关系。
- 图形保持几何化，不直接绘制大树、祠堂或复杂姓氏图腾。
- 小程序图标在 48px 尺寸下仍能清楚辨认。

中文字标使用现代、稳重的黑体结构。书法字体只可用于品牌海报中的少量装饰，不进入功能界面。

## 3. 色彩系统

### 3.1 主色板

| Token | 色值 | 用途 |
|---|---|---|
| `brand-primary` | `#245C4A` | 主按钮、选中状态、核心关系线 |
| `brand-primary-pressed` | `#194537` | 主按钮按下态 |
| `brand-primary-light` | `#E4EFEA` | 轻提示、选中背景、当前分支 |
| `brand-accent` | `#B95338` | 少量强调、提醒、新增标记 |
| `canvas-bg` | `#F7F4EC` | 小程序和家谱画布背景 |
| `surface` | `#FFFEFA` | 卡片、浮层、表单背景 |
| `text-primary` | `#202824` | 标题、正文、成员姓名 |
| `text-secondary` | `#65706A` | 辅助信息、时间、说明 |
| `border` | `#D8DED9` | 分割线、卡片边框、普通关系线 |

主色采用深松绿色，表达生长、稳定和家庭延续；暖米白代替纯白，保留温度；朱砂红只承担点睛和重要提醒，不能大面积铺满页面。

### 3.2 功能色

| 状态 | 色值 | 使用规则 |
|---|---|---|
| 成功 | `#337A57` | 保存成功、审核通过 |
| 提醒 | `#A66B1F` | 待补资料、待审核 |
| 错误 | `#B43D3D` | 删除、冲突、失败 |
| 信息 | `#356A8A` | 帮助、系统说明 |

所有状态必须同时提供图标或文字，不能只依赖颜色表达。

### 3.3 性别与人物状态

- 不用蓝色/粉色区分性别，统一使用中性成员卡。
- 性别仅在确有业务需要时以“男/女/未填写”文字呈现。
- 已故成员使用“故”文字标记和较低饱和度头像边框，不只使用灰色头像。
- 当前视角、当前选中和管理员身份使用不同形状或标签，不只改变颜色。

## 4. 字体与文字层级

用户端采用微信系统默认中文字体，优先保证加载速度和不同设备的一致性。

| 层级 | 字号建议 | 字重 | 使用场景 |
|---|---:|---|---|
| 页面大标题 | 48rpx / 24px | 600 | 家谱名称、关键结果 |
| 页面标题 | 40rpx / 20px | 600 | 页面导航、弹窗标题 |
| 卡片标题 | 36rpx / 18px | 600 | 成员姓名、模块标题 |
| 正文 | 32rpx / 16px | 400 | 表单、说明、按钮 |
| 辅助文字 | 28rpx / 14px | 400 | 日期、关系路径、提示 |
| 标签 | 24rpx / 12px | 500 | 状态标签、角色标签 |

规则：

- 正文默认不低于 32rpx；重要页面不使用大段小字。
- 姓名优先完整显示，空间不足时最多两行，不轻易截断。
- 数字、日期和关系称谓保持较强对比度。
- 支持跟随系统字体放大；关键布局需要在 115% 字号下正常使用。

## 5. 布局与空间

采用 8px 基础栅格，对应小程序常用的 16rpx 节奏。

- 页面左右安全边距：32rpx。
- 模块间距：32—48rpx。
- 卡片内边距：24—32rpx。
- 普通卡片圆角：24rpx。
- 小标签圆角：12rpx。
- 主按钮高度：96rpx，最小点击区域不低于 88rpx。
- 卡片以细边框为主，阴影只用于浮层和悬浮工具栏。

页面背景保持干净，不叠加高对比纹理。需要文化氛围时，可以使用透明度极低的点阵或枝叶线稿，但不得影响文字和图谱连接线。

## 6. 家谱图视觉规范

家谱图是「有谱」最核心的品牌界面，优先级高于首页装饰和运营插图。

### 6.1 全谱模式

全谱模式采用中性画布，不绑定人物视角：

- 按代际从上到下排列，允许切换为从左到右布局。
- 同代成员保持基本对齐，伴侣靠近展示。
- 父母—子女关系使用连续实线。
- 伴侣关系使用较短横向连接线，并在两人之间形成视觉组合。
- 待确认关系使用虚线，并附“待确认”标签。
- 分支较大时支持折叠，折叠点显示“还有 8 人”。
- 默认自动适配可见范围，用户可以拖动和双指缩放。

普通关系线使用中性灰绿色；选中一个成员时，只高亮该成员的直接关系和当前路径，其他分支降低对比度，但仍保持可见。

### 6.2 成员视角模式

- 当前视角成员位于画布视觉中心。
- 使用深松绿描边和柔和外环明确标识当前视角。
- 其他成员显示“父亲、女儿、岳父”等相对称谓。
- 与当前成员的关系路径使用主色高亮。
- 顶部持续显示“从张建国看”，并提供清晰的“返回全谱”。
- 无法可靠推导称谓时展示关系路径，不猜测复杂称谓。

### 6.3 成员节点

成员节点建议采用“头像 + 姓名 + 一行辅助信息”的小卡片：

- 常规宽度约 152—176rpx，随字号适配。
- 头像使用圆形或轻圆角方形，不能过度占据节点。
- 姓名是节点最高信息层级。
- 生卒年份、称谓或状态最多显示一行。
- 当前选中、当前视角、待审核分别使用边框、外环、标签表达。
- 缺少头像时使用统一的线性人物占位图，不使用随机卡通头像。

### 6.4 大家谱降噪

- 默认展示结构，不在每个节点上堆叠操作按钮。
- 操作在点击节点后的浮动人物卡中出现。
- 缩小到一定比例后隐藏头像和辅助信息，只保留姓名。
- 再次缩小时以家庭分支摘要代替单个人物节点。
- 搜索命中后自动定位并高亮，但不强制切换成员视角。

## 7. 图标与插画

### 7.1 图标

- 使用统一的 2px 线性圆角图标。
- 图标轮廓简洁，避免传统花纹和拟物阴影。
- 核心操作必须配文字，如“添加亲属”“邀请家人”，不能只放图标。
- 底部导航建议使用：关系图、家庭成员、个人设置三个语义明确的图标。
- 删除等危险操作不用垃圾桶 emoji，使用规范图标和文字确认。

### 7.2 插画

插画采用少量、低饱和、平面线稿风格，可以使用人物剪影、相册、枝条和连接点等元素。

适用场景：

- 首次创建引导。
- 空家谱、空待办、邀请成功。
- 隐私和数据导出说明。

不在家谱主画布加入大型插画，不让装饰遮挡关系结构。

## 8. 核心组件风格

### 8.1 按钮

- 主按钮：深松绿实底、白字，用于“创建家谱”“添加亲属”“邀请家人”。
- 次按钮：白底、主色描边，用于“稍后补充”“返回全谱”。
- 文字按钮：用于低优先级操作，如“查看全部”。
- 危险按钮：默认使用文字或浅红背景，二次确认后才显示实色危险操作。
- 同一页面最多只有一个主按钮。

### 8.2 卡片

- 主要卡片使用暖白背景和细边框。
- 普通内容不使用悬浮大阴影。
- 家庭动态、待办、邀请等卡片拥有固定信息结构。
- 卡片整块可点击时保留明显按下态。

### 8.3 表单

- 一屏只完成一个主要任务。
- 标签位于输入框上方，不只依赖 placeholder。
- 创建成员时只要求姓名，其他字段渐进补充。
- 关系选择使用图形化关系按钮，不让用户选择数据库式“父亲 ID”。
- 保存成功后明确显示“已保存”，弱网状态显示“正在保存/保存失败”。

### 8.4 浮层

成员人物卡优先使用底部半屏抽屉，保留对家谱上下文的感知。只有高风险确认、权限说明和复杂编辑才使用完整页面。

## 9. 动效与反馈

动效服务于空间关系理解，不用于炫技：

- 切换成员视角时，画布平滑移动到新中心，而不是瞬间跳转。
- 添加亲属后，新节点从关系位置轻微展开，并短暂高亮。
- 返回全谱时平滑缩放到全局适配范围。
- 普通过渡控制在 180—240ms，画布定位可在 280—360ms。
- 保存、邀请和审核结果使用轻量反馈，不使用庆祝彩带等强娱乐动效。
- 尊重系统减少动态效果设置，必要时退化为淡入淡出。

## 10. 文案语气

语气应该像一位耐心的家庭整理助手：直接、温和、不卖弄专业术语。

推荐：

- “添加一位家人”而不是“新增节点”。
- “从他看家谱”而不是“切换中心用户”。
- “关系还不能确定，请管理员确认”而不是“数据校验失败”。
- “邀请家人一起补全”而不是“立即裂变分享”。
- “暂时没有填写”而不是“无数据”。

涉及删除、退出和隐私时使用明确措辞，不用模糊的温情文案弱化风险。

## 11. 关键页面风格

### 11.1 家谱页

- 家谱画布占据屏幕主体。
- 顶部使用暖白色轻量工具栏，提供家谱切换、全谱/成员视角和分享。
- 底部仅保留一个“添加亲属”主操作。
- 缩放、适配和搜索使用小型悬浮工具组。

### 11.2 家庭页

- 比家谱页更接近卡片式信息面板。
- 顶部展示共同维护人数和资料完整度，不做竞争性排行榜。
- 待办优先于动态，邀请家人保持明显但不过度营销。

### 11.3 我的

- 以设置列表为主，视觉安静。
- 家谱切换、最近查看、隐私与数据管理优先。
- openid 等技术信息不直接展示给普通用户。

### 11.4 分享卡片

- 保持品牌主色和暖米白底。
- 只显示家谱名称、分享者称呼、成员数和明确动作。
- 成员视角卡显示“从张建国看李家家谱”。
- 不在卡片中展示完整生日、联系方式、未成年人照片等敏感信息。

## 12. 运营后台风格

运营后台沿用品牌色，但整体更专业、密度更高：

- 背景采用浅灰白，数据卡和表格使用白色。
- 主色仅用于导航选中、主操作和图表主序列。
- 状态色严格对应成功、提醒、错误和信息。
- 家谱关系预览沿用用户端连接线语义，避免运营人员重新学习。
- 敏感信息默认遮罩，查看时要求填写原因并留下审计记录。
- 不使用用户端插画和温情文案，操作语义以准确为先。

## 13. 设计验收原则

每个页面交付前检查：

1. 用户能否在 3 秒内判断当前页面的主要任务。
2. 页面是否只有一个最高优先级操作。
3. 年长用户能否看清文字并准确点击目标。
4. 是否仅靠颜色表达关系或状态。
5. 全谱模式是否在不选择任何成员的情况下成立。
6. 成员视角是否明确显示当前从谁来看，并能一步返回全谱。
7. 家谱缩放后是否仍能识别成员与关系层级。
8. 保存、失败、待审核和权限不足是否有明确反馈。
9. 分享卡片是否泄露不必要的家庭信息。
10. 页面装饰是否影响了人名、连接线或操作识别。

## 14. 一句话设计结论

> 「有谱」要像一本被重新设计过的现代家庭册：温暖但不陈旧，庄重但不沉重，关系复杂但操作简单。
