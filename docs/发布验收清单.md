# 有谱 v1.0.0 发布验收清单

## 自动化与构建

- [ ] 所有 JavaScript 语法检查通过。
- [ ] `node --test tests/*.test.js` 全部通过。
- [ ] 管理端 `pnpm run typecheck` 与生产构建通过。
- [ ] 微信 CLI 预览通过，公共基础库为 3.16.2，`urlCheck: true`。
- [ ] 发布分支、完整提交和 `v1.0.0` 标签已创建。

## 用户端

- [ ] 新用户首屏直接出现“创建家谱”和“接受邀请”，没有永久加载。
- [ ] 管理员可直接维护；member 只能提交申请；viewer 只读。
- [ ] 默认全谱可用；任意人物视角、返回全谱、搜索和折叠分支可用。
- [ ] 500 人/对应关系首次有效渲染 ≤ 2.5 秒，拖动、缩放和切视角无明显卡顿。
- [ ] 邀请默认 30 天/50 次，并可撤销；20 个并发接受不超过次数上限且不重复入组。
- [ ] 管理员转让、退出、归档/恢复正确；最后一名管理员在 active 家谱不能退出。
- [ ] 缺少页面参数、无权限、弱网、服务错误都有安全状态和重试入口。

## 数据与安全

- [ ] 唯一索引全部处于可用状态。
- [ ] 客户端数据库读写全部拒绝，未登录云函数调用全部拒绝。
- [ ] 自关联、重复、跨家谱、父母子女环均被拒绝。
- [ ] 重复写请求返回第一次结果；事务失败不留下半成品。
- [ ] 所有列表均可继续游标分页；超过 500 人明确报错而非静默截断。
- [ ] 图片只能进入本人私有暂存目录；原文件不可直读；短链需家庭权限且只有审核通过后返回。

## 隐私与内容安全

- [ ] 隐私政策主体为公众平台登记全称，服务协议、客服、权限用途可访问。
- [ ] 首次隐私授权、拒绝相册/相机、撤回权限不影响只读浏览。
- [ ] 文本通过/疑似/违规三类结果正确；图片 pending/review/approved/rejected 正确。
- [ ] 人物、头像、家谱描述和邀请均可举报，用户可查看处理结果。
- [ ] 信息导出可用；注销 7 天冷静期可撤销，到期匿名化符合预期。

## 运营后台

- [ ] 未登录、非白名单均不可进入。
- [ ] operator 与 super_admin 权限矩阵正确。
- [ ] 用户/家谱默认脱敏；查看必要内容必须关联工单、填写原因并产生审计。
- [ ] 举报、复核、冻结、注销重试和运营账号操作全部二次确认、要求原因、可审计。
- [ ] 运营端不存在编辑家庭人物关系的入口。

## 平台与运维

- [ ] 独立预发布环境全量验收完成。
- [ ] 生产数据库备份、完整导出、存储清单和恢复演练完成。
- [ ] 错误率、P95、审核积压、资源 80% 和注销失败告警已实测触发。
- [ ] 备案、主体认证、服务类目、隐私指引、客服和审核说明完成。
- [ ] 微信 CLI、真机、体验版各执行一轮并留存验收记录。

所有复选项必须有验收人、时间和证据链接；不接受口头确认。
