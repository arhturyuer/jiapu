# 有谱正式发布运行手册

## 一、硬性发布门槛

以下任何一项未完成，都不得清空生产、上传正式版或提交审核：

1. 已创建独立预发布环境，且小程序、三类云函数、管理端、索引和安全规则均部署在该环境。
2. `miniprogram/config/legal.js` 已替换为微信公众平台登记主体全称，微信客服可用。
3. 内容审核异步回调实测通过，原图不能被客户端直接读取。
4. 预发布自动化、500 人性能、三角色权限、真机和体验版全量验收通过。
5. 生产数据库已创建可恢复时间点/备份，并完成独立数据导出；存储对象清单已导出并校验数量。
6. 备份恢复演练已在非生产环境成功，集合数、记录数、抽样哈希和对象数量一致。
7. 备案、主体认证、服务类目、隐私指引、客服和审核说明均已在微信公众平台完成。

## 二、预发布部署

1. 在 CloudBase 控制台创建预发布环境并记录环境 ID、套餐、地域和负责人。
2. 将 `miniprogram/config/env.js` 的 staging ID 填好并设为 active。
3. 应用集合、索引、数据库/存储/函数安全规则。
4. 配置三个云函数的 Nodejs20.19、超时、内存、环境变量、OpenAPI 权限和两个定时触发器。
5. 执行 bootstrap，创建 Auth v2 初始用户与运营白名单。
6. 构建并部署运营后台静态站点。
7. 配置内容审核回调、告警和定时任务。
8. 执行 `TARGET_ENV_ID=<预发布ID> ./deployment/preflight.sh`。
9. 完成 `docs/发布验收清单.md`，上传体验版并真机复验。

## 三、生产备份与恢复演练

在变更前记录生产环境 `cloud1-d5gs5yj4l283d9c6d` 的所有集合、记录数、索引、存储对象键/大小/ETag、云函数版本和环境变量（密钥只记录密钥版本，不导出明文）。

数据库备份必须同时满足：

- CloudBase 控制台存在本次变更前可选择的恢复点；
- 另有一次完整集合导出，保存在与生产环境故障域隔离的位置；
- 每日备份保留 30 天。若当前 CloudBase 套餐/产品界面只提供更短保留期，必须通过定时导出到独立存储或升级套餐补齐，不能用 `backup_manifests` 代替真实备份。

`backup_manifests` 只记录每日集合计数，用于发现备份缺口，不包含业务数据。

恢复演练要在新集合或独立环境完成，禁止覆盖生产。至少核对：全部集合存在、总记录数一致、用户/家谱/人物/关系抽样可关联、媒体对象数量一致、唯一索引可重建、运营账号仍需重新校验白名单。

## 四、生产初始化与灰度切换

1. 冻结生产写入窗口，确认无正在处理的注销、归档删除或内容审核任务。
2. 再次生成数据库恢复点、数据导出和存储清单，双人确认可恢复。
3. 仅在上一步完成后清空旧业务集合；不要删除云环境本身。
4. 部署三个新云函数，bootstrap 集合，创建索引并应用安全规则。
5. 保留旧 `familyFunctions` 但仅作为上一版小程序回滚入口。
6. 用生产测试账号执行最短冒烟：新建家谱、邀请三角色、成员申请、管理员审核、全谱/视角、举报、媒体审核、归档恢复、注销取消。
7. 将小程序环境切回 production，运行生产预检：`ALLOW_PRODUCTION=1 TARGET_ENV_ID=cloud1-d5gs5yj4l283d9c6d ./deployment/preflight.sh`。
8. 上传 `v1.0.0`，提交微信审核；审核通过后由负责人手动发布。
9. 观察 30 分钟核心指标，再逐步通知家庭群使用。

## 五、监控与告警

三类云函数日志字段固定为 `requestId / actorId / action / durationMs / success / resultCode`。`actorId` 是不可逆内部标识，不记录姓名、简介、邀请令牌、图片地址或原始 openid。

配置以下告警：

| 指标 | 阈值 | 处置 |
|---|---|---|
| 5 分钟错误率 | > 2% | 暂停发布/回滚客户端，按 requestId 排查 |
| API P95 | > 2 秒 | 检查慢查询、索引、内容审核与资源配额 |
| 审核队列最老任务 | > 30 分钟 | 检查回调，切人工复核 |
| 数据库/函数/存储用量 | > 80% | 扩容并检查异常增长 |
| 注销 `failed` | 任意新增 | 运营工单重试并记录原因 |
| 家谱 `deleting` | 超过 30 分钟 | `youpuJobs` 自动续跑，仍失败则人工介入 |

## 六、回滚

- 小程序异常：回滚上一审核版本；不要回滚已经成功提交的数据库事务。
- 新用户 API 异常：将上一版小程序继续指向旧 `familyFunctions`，保留新集合现场；修复后从预发布重新验收。
- 管理端异常：回滚静态托管上一版本，不允许运营人员直接改数据库。
- 定时任务异常：先停触发器，再保留失败状态与 requestId，修复后幂等重跑。
- 数据损坏：停止写入，复制生产现场，优先在新集合恢复并校验后切换；禁止直接覆盖唯一可用副本。

生产稳定观察期不少于 7 天。确认无旧版调用后再应用 `function-rules-after-stabilization.json` 并停用旧函数。
